Docker Learning Notes

Core Dockerfile concepts learned:

1.  FROM python:3.13-slim

-   Chooses the base image (Linux + Python installed).
-   Every Dockerfile starts from some base image.

2.  WORKDIR /containerisedApp

-   Sets the working directory inside the container.
-   Equivalent to ‘cd’ in a terminal.
-   Creates the directory if needed.

3.  COPY requirements.txt ./

-   COPY
-   First argument is on your computer.
-   Second argument is inside the container.
-   ‘.’ means current working directory (WORKDIR).

requirements.txt: - Lists dependencies (e.g. pandas). - Copy separately
to improve Docker build caching.

4.  RUN pip install –no-cache-dir -r requirements.txt

-   RUN executes during image build.
-   Installs dependencies.
-   –no-cache-dir is pip’s cache, not Docker’s cache.

RUN vs CMD: - RUN = build time. - CMD = container startup.

5.  COPY turbineProcessing.py telemetry_data_in.csv ./

-   Copies application files into container.
-   Can copy specific files instead of everything.
-   COPY . . copies the whole project.

6.  CMD [“python”,“turbineProcessing.py”]

-   Default command when container starts.
-   Equivalent to running python turbineProcessing.py.

Image vs Container: Dockerfile -> build -> Image -> run -> Container

Image = template. Container = running instance. One image can create
many containers.

Useful commands: docker build -t turbine-app . docker run –rm
turbine-app docker run –rm –name turbine-container turbine-app docker
images docker ps docker ps -a docker rm docker rmi docker system prune

–rm: Deletes the container after it exits. Image remains.

–name: Assigns your own container name.

Docker build cache: Docker caches layers. Copying requirements first
avoids reinstalling packages if only code changes.

Images are immutable: Edit code -> rebuild image.

Interactive shell: docker run -it python:3.13-slim bash

Volumes: Share files between host and container: docker run -v
HOSTDIR:/containerisedApp image

Environment variables: docker run -e API_KEY=value image

Ports: docker run -p 5000:5000 image

Docker Compose: Used for multi-container apps. docker compose up

Mental model: A container is a tiny disposable Linux computer. Describe
it with a Dockerfile, rebuild instead of manually editing running
containers.
